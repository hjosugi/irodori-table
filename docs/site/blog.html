Database guide

Irodori Table のDB別ガイド

Irodori Table は、軽量なデスクトップ体験と広いデータソース対応を 両立させるためのオープンソースDBワークベンチです。このページでは、 どのDBで何を試せるか、どの公式情報を追えばよいかをまとめます。

Overview

ひとつの作業台で、DBごとの違いを隠しすぎない。

Irodori Table は、PostgreSQL や MySQL のような日常的なSQL DBから、 MongoDB、TimescaleDB、DuckDB、クラウドWarehouse、Graph、Vector DBまで 同じアプリで扱えることを目標にしています。ただし、すべてを無理に 「表」に押し込むのではなく、DBごとの強みを試せるサンプルとUI契約を 分けて育てます。

現在の位置づけ

Development preview です。接続・実行・メタデータ取得が進んでいるDBと、 登録済みだが専用UXや検証がこれからのDBを明確に分けています。

Samples

DB固有機能はサンプルプロジェクトで確認する。

汎用の疎通確認は make db-verify DB=<engine> で実行します。 DB固有の機能確認は samples/projects/<engine>/ のクエリ集を Irodori Tableで開いて、1文ずつ実行します。 

make db-up DB=postgres
# samples/projects/postgres/queries.sql を開いて実行
make db-down DB=postgres

node tools/docs/db-feature-samples.mjs

サンプルと公式リンクの一覧は samples/db-feature-samples.json に集約しています。

Runnable samples

PostgreSQL

MySQL

JSON関数、外部キー、window関数、EXPLAIN FORMAT=JSONを確認します。

MySQL Docs samples/projects/mysql/queries.sql

MariaDB

MySQL wireで接続し、JSON_VALID、JSON_VALUE、再帰CTEを試します。

MariaDB Docs samples/projects/mariadb/queries.sql

SQLite

サーバなしで、JSON、FTS5、PRAGMA、外部キー制約を確認できます。

SQLite Docs samples/projects/sqlite/queries.sql

DuckDB

ローカル分析向けに、struct、summarize、window、EXPLAINを試します。

DuckDB Docs samples/projects/duckdb/queries.sql

SQL Server

TDS経由で、T-SQL JSON、identity、TOP、OFFSET/FETCHを確認します。

SQL Server Docs samples/projects/sqlserver/queries.sql

Oracle Database

Thin TNSで接続し、SQL/JSON、分析関数、DBMS_XPLANを試します。

Oracle Docs samples/projects/oracle/queries.sql

MongoDB
