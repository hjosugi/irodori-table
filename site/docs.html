Overview

現在の位置づけ

Irodori Table は、軽量なオープンソース DB ワークベンチを目指す開発中プロジェクトです。 Rust コア、Tauri デスクトップ、React UI、CodeMirror 6 エディタで構成しています。 現時点では安定リリース前の development preview です。

公開ステータス

日常利用できる完成版としてではなく、接続エンジン、クエリ実行、結果表示、 補完、拡張基盤を段階的に検証している状態として扱ってください。

Quick start

ローカルで起動する

デスクトップアプリは apps/desktop にあります。ブラウザ preview は Tauri API がない場合に mock shell として動きます。実接続やローカル OS 機能は Tauri shell で確認します。 

cd apps/desktop
npm install
npm run dev

型生成、TypeScript、Vite build まで確認する場合は次を実行します。

cd apps/desktop
npm run build

GitHub Pages の公式サイトだけ確認する場合は、リポジトリ root から次で表示できます。

python3 -m http.server 8080 --directory site

Connections

接続と対応データソース

実装済みの接続層は SQL だけでなく document / key-value / warehouse 系にも広げています。 UI と metadata の品質はエンジンごとに差があるため、詳細な検証状況は implementation progress を正とします。

PostgreSQL MySQL MariaDB SQL Server Oracle SQLite DuckDB MongoDB ClickHouse Snowflake BigQuery Redis Cassandra

Query editor

クエリを書く、実行する

エディタは CodeMirror 6 を使い、エンジンに応じた SQL dialect、Vim mode、format action、 comment toggle、Run Current / selection 実行の導線を持ちます。クエリ履歴は接続ごとに残り、 実行結果には elapsed time、row count、truncated status が表示されます。 

select c.id, c.name, sum(o.total) as lifetime_value
from customers c
join orders o on o.customer_id = c.id
where o.created_at >= now() - interval '90 days'
group by c.id, c.name;

Completion

軽量な schema-aware completion

補完は外部サービスや AI に依存しません。現在の文、カーソル周辺、ローカル metadata を浅く見て、 schema、table、view、column、routine、keyword、alias、FK join snippet を候補にします。 巨大なインデックスや重い全体解析ではなく、入力を止めないことを優先しています。

現在できること

schema. から relation、alias. から column、JOIN の relation 位置で外部キーに基づく table alias on left.col = right.col 形式の候補を出します。

Results

結果グリッドと export

結果はストリーミングと上限付き取得を前提にしています。デフォルトでは巨大 result set を丸ごと メモリに積まず、truncated flag で上限到達を示します。UI 側は row virtualization で表示行を絞り、 CSV export を提供します。

Security

秘密情報と接続経路

パスワードは UI で session-only として扱い、永続化対象から外します。Rust 側には secure-store、 redaction、audit log、direct / SSH / SOCKS5 / HTTP CONNECT / multi-hop proxy の設計があります。 UI への完全な接続は段階的に進めています。

Extensions

拡張 SDK

irodori.extension.json、TypeScript SDK、typed API、dev fixture、テーマ、結果ビュー、driver、 dialect API を段階的に公開する方針です。公式テンプレートは MIT OR 0BSD に揃えます。
