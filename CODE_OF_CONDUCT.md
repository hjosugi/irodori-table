# Code Of Conduct

Irodori Table contributors are expected to keep project spaces practical,
respectful, and focused on the work.

## Expected Behavior

- Be direct without being hostile.
- Assume good intent, but correct inaccurate technical claims with evidence.
- Keep issues and PRs scoped to reproducible facts, design tradeoffs, and next
  actions.
- Respect clean-room and licensing boundaries even when a shortcut looks useful.
- Avoid sharing secrets, private customer data, or non-public third-party
  material in public project spaces.

## Unacceptable Behavior

- Harassment, insults, threats, or sustained personal attacks.
- Publishing private information without permission.
- Introducing copied code, assets, screenshots, or text that violates the
  project's licensing and clean-room rules.
- Repeatedly derailing issues or PRs after maintainers have set scope.

## Enforcement

Maintainers may edit or hide comments, close issues, block participants, or
escalate through GitHub moderation when needed. Security reports should follow
`SECURITY.md` rather than public issue discussion.
